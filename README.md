## 📊 预测架构

### 12步预测流程

```
① AI预测 + 竞彩赔率 交叉验证
② 球员状态评分 (两队100分制+分差解读)
③ 战术对抗分析 (风格相克)
④ 主场优势量化 (5维25分)
⑤ 天气+海拔影响
⑥ 定位球+完整比分分布表
⑦ 裁判尺度分析
⑧ 轮换+心理形势
⑨ 博彩数学验证 (Kelly+find_edge)
⑩ 防守体系评分 (门将+防线100分制)
⑪ 周易玄学 (天干地支+纳音+奇门, ±35分降权)
⑫ 投注建议 (方向优先, 比分向上偏, 强队2球差起步)
```

### 🛡️ 强制规则 (11条)

| # | 规则 | 为什么 |
|:---:|------|------|
| 1 | 比分覆盖**0-4球+全区间 + 强队丢1球**(2:1/3:1) + **双进球**(2:2) | 德国7:1=先1:1再屠杀 |
| 2 | betting工具**每次必调**≥2个(kelly+find_edge) | 不做纯主观判断 |
| 3 | 展示**完整低赔Top8**，不替用户做选择 | 透明度 |
| 4 | 天气+裁判**必须实际搜索** | 空数据=假分析 |
| 5 | 竞彩 vs AI vs Polymarket**三方对比** | 找赔率偏差 |
| 6 | 动态比例：strong→60:40, narrow→40:60, moderate→50:50 | 置信度驱动 |
| 7 | confidence字段**必须读取标注** | strong=可靠信号 |
| 8 | 防守+玄学**必须跑** | 门将+防线可改变战局 |
| 9 | 预选赛数据**对手<FIFA60→打折×0.5** | 突尼斯非洲零封→世界杯丢5球 |
| 10 | 单因果链需**≥2个独立信号**才采信 | 库卢缺→瑞典死=1信号→不采信 |
| 11 | 玄学**±70→±35**，仅辅助 | 6/15已申合翻车教训 |

### 🧠 反线性推理

| 规则 | 公式 |
|------|------|
| 预选赛打折 | 对手均<FIFA60 → 防守/进攻纪录×0.5 |
| 断链验证 | 单因果需≥2独立信号 |
| 玄学降权 | ±70→±35 |
| 冷门≠必押 | 冷门信号+2额外信号才全押 |
| 屠杀分级 | AI>90%→可押屠杀；70-90%→只到3:0 |
| 比分多样 | 必须含2:1/3:1/4:1和2:2 |
| 赛事统计 | 每轮先拉本届已有数据 |

## 🚀 安装

```bash
# 1. 克隆
git clone git@github.com:li3jia4hao5-hue/jc-mcp.git
cd jc-mcp && npm install && npm run build

# 2. 注册全部MCP服务
claude mcp add jc-mcp -- node "$(pwd)/dist/index.js"
claude mcp add wc26 -- npx -y wc26-mcp
claude mcp add onside-football -- npx -y onside-football-mcp

# 3. 安装预测市场+博彩工具
pip install git+https://github.com/machina-sports/sports-skills.git

# 4. 复制预测系统到你的项目
cp CLAUDE.md /你的项目/CLAUDE.md
```
